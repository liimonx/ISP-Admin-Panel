🚀 QUICK START (5 MINUTES):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Copy production environment:
   cp .env.prod.example .env.prod

2. Edit configuration (CRITICAL - Change all defaults!):
   nano .env.prod

   REQUIRED CHANGES:
   ✓ SECRET_KEY - Generate strong random key
   ✓ POSTGRES_PASSWORD - Set secure database password  
   ✓ REDIS_PASSWORD - Set secure Redis password
   ✓ ALLOWED_HOSTS - Set your domain names
   ✓ MAIN_ROUTER_IP - Configure your router IP
   ✓ EMAIL settings - Configure email provider
   ✓ Payment keys - If using payments

3. Setup SSL certificates:
   sudo certbot certonly --standalone -d yourdomain.com
   mkdir -p ssl
   sudo cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem ssl/cert.pem
   sudo cp /etc/letsencrypt/live/yourdomain.com/privkey.pem ssl/key.pem
   sudo chown -R $USER:$USER ssl/

4. Run deployment:
   chmod +x deploy.sh
   ./deploy.sh

5. Verify deployment:
   ./scripts/health-check.sh
   docker-compose -f docker-compose.prod.yml ps

  □ EMAIL_HOST_PASSWORD → App-specific password (not main password)

  □ CSRF_COOKIE_SECURE  → True

🚨 TROUBLESHOOTING QUICK LINKS:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Services won't start?
  → Check .env.prod configuration
  → View logs: docker-compose -f docker-compose.prod.yml logs
  → Check disk space: df -h

Database connection failed?
  → Verify database is running: docker-compose -f docker-compose.prod.yml ps db
  → Check credentials in .env.prod
  → View db logs: docker-compose -f docker-compose.prod.yml logs db

High memory usage?
  → docker stats (identify container)
  → docker-compose -f docker-compose.prod.yml restart [service]
  → Check for memory leaks in logs

SSL certificate issues?
  → Verify cert: openssl x509 -in ssl/cert.pem -text -noout
  → Renew: sudo certbot renew
  → Copy to ssl/: cp /etc/letsencrypt/live/domain/fullchain.pem ssl/cert.pem
